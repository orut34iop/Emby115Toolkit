# EmbyLlmTool - 项目概述

## 项目目标

基于 Emby115Toolkit 的现有功能，引入大模型技术（LLM）进行重构，采用工作流编排方式将 7 个独立功能页串联起来，实现从 "手动逐个操作" 到 "一键自动化执行" 的升级。

## V2 当前命令

推荐动作名：

```bash
python main.py --action build_symlink_workspace
```

兼容别名：

```bash
python main.py --action scan_and_link
```

启动 WebUI 后端骨架：

```bash
python main.py --serve-web
```

测试 TMDB 配置：

```bash
python main.py --action test_tmdb_config --config emby115_v2.config.json
```

测试 LLM 配置：

```bash
python main.py --action test_llm_config --config emby115_v2.config.json
```

元数据刮削 dry-run：

```bash
python main.py --action scrape_metadata --config emby115_v2.config.json --dry-run
```

构建网盘已刮削媒体库 dry-run：

```bash
python main.py --action build_cloud_scraped_library --config emby115_v2.config.json --dry-run
```

该流程使用 `path_pairs`，其中 `source` 是本地 symlink 工作区，`target` 是 D 盘新建的 organized 媒体库目录。真实执行默认会在复制非 symlink 文件后固定等待 60 分钟，再移动 symlink 指向的真实视频文件；可通过 `--cloud-wait-minutes` 调整等待时间，或用 `--cloud-metadata-only` 只执行阶段 A。

CloudDrive2 上传任务探测：

```bash
python main.py --action test_clouddrive2_upload_wait --source C:\working-emby\movies --target D:\115open\tmp\organized-probe --cd2-api-token YOUR_CD2_TOKEN
```

该命令会在 `target\.emby115_cd2_probe` 下写入一个小探测文件，轮询 CloudDrive2 gRPC 上传任务列表，确认挂载盘写入是否能作为 `Mount` 上传任务被观测到。验证通过后，可在真实 `build_cloud_scraped_library` 中使用：

```bash
python main.py --action build_cloud_scraped_library --config emby115_v2.config.json --cloud-upload-wait-strategy clouddrive2_or_fixed
```

`clouddrive2_or_fixed` 会优先等待 CloudDrive2 上传任务完成；如果 API 不可用或没有观测到匹配任务，则回退到 `--cloud-wait-minutes` 的固定等待。严格模式 `clouddrive2` 不回退，无法确认上传完成时会在移动真实视频前失败。

WebUI 的“执行完整流程”当前是浏览器端编排能力，不对应新的 CLI action；CLI 仍按单个 action 调用。完整流程和单独“刮削媒体元数据”队列的取消都采用安全协作模式：通过 `/v1/runs/{run_id}/cancel` 请求当前后台 run 停止，并停止启动后续步骤，不采用强杀线程。

局域网访问必须设置 access token：

```bash
python main.py --serve-web --host 0.0.0.0 --access-token YOUR_TOKEN
```

## 源项目

- 项目名：Emby115Toolkit
- 路径：C:\dev\Emby115Toolkit
- 当前功能：7个独立的GUI功能页（tkinter + PyQt5双前端）
- 后端模块：autosync/（文件系统操作）、emby/（Emby API操作）
- 配置：YAML单例Config

## V2 代码结构

- `emby115_v2/context.py` — 统一 Context Object
- `emby115_v2/cli.py` — CLI facade
- `emby115_v2/web/` — WebUI backend facade
- `emby115_v2/workflow/` — 工作流执行器
- `emby115_v2/services/` — 核心服务层
- `emby115_v2/reports/` — HTML/JSON 报告生成
- `emby115_v2/config_store.py` — V2 独立 JSON 配置读写，默认文件 `emby115_v2.config.json`
